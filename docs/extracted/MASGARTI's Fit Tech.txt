The trainer view explicitly states "VIEW ONLY — Data owned by client." Export and delete options are never hidden. The user should feel sovereign over their information at all times. PRINCIPLE 2: 10-Second Rule for Data Entry Logging a single set must be completable in under 10 seconds. This means: large tap targets, numeric input with increment buttons (not keyboard), pre-filled values from previous session, and single-tap confirmation. The RPE scale uses plain language ("How hard?") not just numbers. PRINCIPLE 3: Context at Point of Decision When logging bench press, the user should see their previous best and estimated 1RM without navigating away. This is the key differentiator — the app surfaces decision-support data where and when it's needed, not buried in a separate analytics section. PRINCIPLE 4: Progressive Disclosure The PARQ is not blocking. Workout logging works from day one. Advanced features (sharing, access control) are discoverable but not in the way. The app grows with the user — a Gen Z user logging solo workouts sees a simple tool; an HNI with a coaching team sees a comprehensive data hub. PRINCIPLE 5: Accessibility Across Ages Minimum touch target: 44×44px. Body text: 14px minimum. High contrast mode ready. No icon-only navigation — all nav items have text labels. No swipe-to-delete or gesture-only interactions. Every action has an explicit button. COLOUR DIRECTION (For Hi-Fi Phase) Primary: Monochrome (black/white/greys) — signals seriousness, not gamification. Accent: Single colour for progress/positive states (suggest deep teal or forest green). Alert: Warm amber for PARQ flags and constraints. Avoid: Bright fitness app colours (neon green, electric blue) — this is a professional tool, not a game.
